Spring事物不起作用原因

发表于 2018-04-27 | 分类于 笔记 | 阅读次数: ℃
字数统计: 795 | 阅读时长 ≈ 4

问题描述

最近更换项目Spring框架javaBean的注入方式，由之前的XML配置换成注解扫描，在测试service层事物时，发现事物失效不起作用，先上如下配置文件。

transaction.xml

仔细检查transaction.xml文件后，再联想之前xml注入javaBean事物都是起作用的。立马考虑是否问题出现在注入方式上。检查spring配置文件后，才发现扫描配置都配置在了springmvc.xml文件中，如下所示。

springmvc.xml

结果测试发现，org.springframework.web.servlet.DispatcherServlet读取springmvc.xml文件，由springmvc去扫描controller层，service层，dao层的javaBean导致事物不起作用，将service层，dao层的扫描交给spring容器扫描，事物就又起作用了。
具体原因也不太清楚，真是自己胡乱挖坑。不过具体原因可以参考 spring,springmcv容器之间的关系，以及注解扫描
